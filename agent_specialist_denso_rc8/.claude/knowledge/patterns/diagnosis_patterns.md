# diagnosis_patterns.md
# 故障诊断判断规则库
# 适用：Task 5（故障诊断与报警分析）

---

## [PAT-020] 根因分层分析规则
**适用场景**：所有故障诊断
**判断规则**：收到错误代码后，按以下顺序排查：
  1. 硬件层（接线、传感器、机械）
  2. 配置层（参数、地址、模式设置）
  3. 程序层（逻辑错误、时序问题）
  4. 通信层（网络、协议、超时）
  不可跳层，不可在未排除上层问题时直接跳到程序层
**例外条件**：有明确历史记录（mistakes/log.md）的已知错误，可直接对照解法
**置信度**：高
**来源任务**：框架初始化
---

## [PAT-021] 可能原因分级输出规则
**适用场景**：故障诊断输出
**判断规则**：输出可能原因时，必须按概率分级：
  P1（最可能，>60%）→ P2（次可能，20-60%）→ P3（低概率，<20%）
  每级附带：判断依据 + 验证方法 + 解决方案
**例外条件**：已在 mistakes/log.md 确认的已知故障，直接输出 P1 + 解法
**置信度**：高
**来源任务**：框架初始化
---

## [PAT-022] 安全优先停机规则
**适用场景**：涉及机械运动的故障诊断
**判断规则**：故障排查步骤的第一步永远是：确认设备已安全停止（E-STOP已触发/电机已断电），才能开始后续排查步骤
**例外条件**：纯软件/通信故障不需要停机，但必须确认无运动风险
**置信度**：高
**来源任务**：框架初始化
---

## [PAT-023] 历史记录优先规则
**适用场景**：收到错误代码或故障描述时
**判断规则**：在分析前先检索 mistakes/log.md 中是否有相同错误代码或相似场景的记录；有记录时优先使用已验证的解法，并注明"已知问题"
**例外条件**：无
**置信度**：高
**来源任务**：框架初始化
