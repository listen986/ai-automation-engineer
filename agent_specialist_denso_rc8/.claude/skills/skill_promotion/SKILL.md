# SKILL: Skill Promotion — 技能自动提升
# 来源：Voyager (Wang et al., 2023) — 技能库自动积累
# 核心原理：将反复成功的解法提炼为可复用技能，写入程序记忆

---

## 核心原理

Voyager 在 Minecraft 中的突破：不只完成任务，
每次成功后把解法提炼为 JavaScript 技能写入库，
下次遇到相似任务直接调用，而非从头推理。

我们的实现：成功解法 → 使用次数计数 → 
达到阈值 → 起草 SKILL.md → 人工确认 → 合并进技能库

---

## 触发条件

当以下条件同时满足时，触发技能提升评估：
1. 某个解法/模式被**成功使用 3 次或以上**
2. 每次使用结果都是成功的（无 ✗ 校正）
3. 该解法具有**通用性**（不只适用于单一设备/场景）

---

## 候选技能追踪

在 `memory/core/CORE.md` 的"Voyager候选"栏记录：
```
解法描述：___
使用次数：1/2/3（达到3触发评估）
使用场景：任务NNN, 任务NNN, 任务NNN
成功率：___
```

---

## 技能提升评估流程

### Step 1：通用性评估
```markdown
## 技能提升评估

**候选解法**：___
**使用记录**：SESSION-NNN × 3
**评估问题**：
  1. 这个解法适用于多少种不同场景？（<2种 → 不提升）
  2. 它依赖于特定设备型号吗？（强依赖 → 不提升或加条件）
  3. 用文字能清楚描述执行步骤吗？（否 → 不提升）
  4. 相比现有 SKILL，它提供了新的覆盖范围吗？（否 → 不提升）

**结论**：提升 / 不提升（原因：___）
```

### Step 2：起草新 SKILL.md
如果评估通过，起草 SKILL 草稿：
```markdown
---
name: [技能名称]
description: [触发条件描述，包含具体场景关键词]
origin: Voyager自动提升 · SESSION-NNN,NNN,NNN · 日期
status: 草稿（待人工确认）
---

# [技能名称]

## 触发条件
[从哪些用户描述中触发]

## 核心解法
[提炼的通用步骤]

## 适用条件
[什么情况适用]

## 不适用条件
[什么情况不适用]

## 示例
[从成功任务中提取的典型示例]

## 来源
[SESSION-NNN 链接]
```

### Step 3：等待人工确认
将草稿保存为 `skills/[名称]/SKILL.draft.md`
在会话中提示用户：
```
💡 Voyager 技能提升建议
解法"___"已成功使用3次。
已起草技能文件：skills/[名称]/SKILL.draft.md
请审阅后决定是否正式加入技能库。
```

### Step 4：人工确认后
- 将 `SKILL.draft.md` 重命名为 `SKILL.md`
- 更新 `memory/core/CORE.md` 中"已提升Voyager技能数"
- 更新 `wiki/index.md` 的 patterns/ 索引
- 写入 `knowledge/episodic/session_log.md`

---

## 不应提升为技能的情况

- 设备强依赖（只适用于一个控制器型号）
- 一次性解法（针对特殊边界条件）
- 与现有 SKILL 高度重叠（<20% 新内容）
- 成功率低于100%（有失败记录则等待更多数据）
