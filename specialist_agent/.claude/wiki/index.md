## Wiki 与 agent_ref 的区别

| | agent_ref/ | wiki/ |
|---|---|---|
| 内容 | 厂商原文，完整手册 | LLM综合，提炼要点 |

| 读取时机 | 执行具体指令时精确查阅 | 任务开始时快速定向 |

## Wiki 写作规范

每个页面必须包含：

```markdown
# [[页面名]] — 标题

**摘要**：一句话（<50字）
**来源**：agent_ref/[文件] | knowledge/patterns/[文件] | 任务经验
**相关页面**：[[页面A]] · [[页面B]]
**最后更新**：YYYY-MM-DD

---

## 核心内容
[LLM综合编写，提炼要点，不抄原文]

## 关键判断规则
[从 patterns/ 提炼，可直接使用]

## 常见错误
[从 mistakes/ 提炼，附根因]

## 待解问题
[来自 open_questions/，驱动继续探索]
```

## Lint 检查项

每10次任务或发现矛盾时执行：
- [ ] 页面间是否有矛盾内容？
- [ ] 有无孤立页面（无被引用）？
- [ ] 有无内容版本低于当前 agent_ref？
- [ ] 有无错误根因已解决但页面未更新？

发现问题记录至 `knowledge/lint/issues.md`
