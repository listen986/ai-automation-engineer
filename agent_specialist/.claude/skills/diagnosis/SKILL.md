# SKILL: Fault Diagnosis — 故障诊断与报警分析
# Task 5

---

## 触发条件
- "错误代码 XXXX" / "报警 XXXX" / "机器停了" / "设备报错"
- 用户描述设备异常行为或提供错误代码

---

## 执行前准备（强制）

### Step 1：搜索历史记录
```
先检索 knowledge/mistakes/log.md 中是否有：
  - 相同错误代码的记录
  - 相同设备的相似故障记录
有记录 → 优先输出已验证的解法，标注"已知问题"
无记录 → 进入标准诊断流程
```

### Step 2：识别设备与错误
```
□ 设备品牌和型号
□ 错误代码（精确）
□ 报警描述文字
□ 发生场景（正常运行中/上电时/特定动作时）
□ 频率（偶发/必现/首次）
□ 最近是否有改动（程序/配置/硬件）
```

### Step 3：查阅手册
从 agent_ref/ 加载对应设备的错误代码手册，查找确切含义。

## 分层排查原则（来自 PAT-020）

```
Layer 1 硬件层：接线松动？传感器损坏？机械干涉？
  ↓ 排除后
Layer 2 配置层：参数设置？地址配置？模式选择？
  ↓ 排除后
Layer 3 程序层：逻辑错误？时序问题？变量溢出？
  ↓ 排除后
Layer 4 通信层：网络连接？超时设置？协议配置？
```

## 特殊情况处理

### 无法在手册中找到错误代码
```markdown
[低置信] 未能在 agent_ref/ 中找到此错误代码。
可能原因：
1. 手册版本不匹配（当前手册版本：___）
2. 错误代码为子代码，需要查看完整错误描述
建议：联系厂商技术支持，提供完整错误信息
```

### 间歇性故障（偶发）
优先考虑：
1. 接线接触不良（振动导致）
2. 噪音干扰（信号线与动力线未分离）
3. 散热问题（温度相关）
4. 电源波动

## 诊断完成后：触发经验萃取

如果此次诊断有新发现：
- 确认的根因 → 写入 knowledge/mistakes/log.md
- 可复用的诊断规则 → 写入 knowledge/patterns/diagnosis_patterns.md
- 未解决的问题 → 写入 knowledge/open_questions/pending.md
