# 工业自动化工程底层原理
# 稳定层：这些原理不随项目变化，人工维护

---

## 1. 工业自动化系统层级模型

```
Level 4: ERP / MES           企业资源计划 / 制造执行系统
Level 3: SCADA / DCS         监控与数据采集 / 分布式控制
Level 2: HMI / Historian     人机界面 / 历史数据
Level 1: PLC / Controller    可编程控制器 / 机器人控制器
Level 0: Field Devices       传感器 / 执行器 / 机器人 / 视觉
```

设计原则：上层依赖下层，下层不感知上层。
通信向上聚合，控制向下传递。

## 2. 安全优先原则（Safety First）

### 安全完整性等级 SIL
- SIL 1：低风险，单一安全功能
- SIL 2：中风险，制造业常见等级
- SIL 3：高风险，需冗余设计

### E-STOP 设计原则
- E-STOP 必须硬线连接，不得依赖软件逻辑
- E-STOP 触发后，必须手动复位才能重新启动
- 安全回路必须在所有其他逻辑之前执行

### 互锁逻辑设计原则
- 先确认安全条件，再允许动作
- 任何动作的允许条件必须包含：设备就绪 + 安全区域清空 + 上游完成信号
- 禁止用 Timer 代替实际完成信号

## 3. 工业通信协议选型原则

| 协议 | 适用场景 | 典型延迟 | 主要厂商 |
|------|---------|---------|---------|
| EtherNet/IP | 通用工业以太网，PLC↔Robot↔HMI | 1-10ms | Rockwell, OMRON |
| PROFINET | 西门子生态，实时性强 | <1ms (IRT) | Siemens |
| Modbus TCP | 简单设备集成，传感器读取 | 10-100ms | 通用 |
| EtherCAT | 高速运动控制，伺服同步 | <100μs | Beckhoff |
| OPC-UA | 跨平台数据交换，MES集成 | 不定 | 通用 |
| IO-Link | 传感器层智能通信 | - | 通用 |

**选型规则**：
- 实时运动控制 → EtherCAT 或 PROFINET IRT
- 通用设备集成 → EtherNet/IP 或 PROFINET RT
- 传感器数据采集 → Modbus TCP 或 IO-Link
- 系统间数据交换 → OPC-UA

## 4. 机器人运动控制基础

### 坐标系层级
```
World Frame → Base Frame → Tool Frame → Work Object Frame
```
- 编程时优先使用 Work Object Frame，便于产品换型
- Tool Frame 必须经过标定才能保证精度

### 运动类型
- PTP（Point to Point）：轴空间插补，速度最快，路径不可预测
- LIN（Linear）：笛卡尔空间直线插补，路径可控
- CIRC（Circular）：圆弧插补，需要辅助点

### 奇异点处理
- 奇异点附近速度骤降，需降速或绕行
- 腕部奇异：J4/J6 共线（关节5接近0度）
- 肘部奇异：机器人完全伸展
- 肩部奇异：J1 轴与 TCP 共线

## 5. PLC 编程基础原则

### 扫描周期原则
- PLC 顺序扫描，每个扫描周期读取输入 → 执行程序 → 更新输出
- 典型扫描时间：1-10ms
- 高速响应（<1ms）需使用中断或高速计数器

### 梯形图设计原则
- 线圈（输出）只能在最右侧
- 同一线圈不得在多处驱动（使用 SET/RESET 指令代替）
- 安全互锁触点放在最左侧（最先执行）

### 结构化文本 ST 设计原则
- 使用功能块（FB）封装可复用逻辑
- 状态机（State Machine）处理复杂顺序逻辑
- 变量命名：前缀标注类型（b=BOOL, i=INT, r=REAL, s=STRING）

## 6. 视觉系统集成原则

### 触发模式
- 硬触发（硬件信号）：实时性高，适合高速产线
- 软触发（通讯指令）：灵活，适合低速场景

### 坐标变换
- 视觉坐标（像素）→ 机器人坐标（mm）需要标定矩阵
- 标定方法：9点标定法（3×3网格）或更多点
- 标定精度直接决定定位精度

### 输出数据格式
- 位置输出：X, Y, θ（旋转角）
- 判断输出：OK/NG + 原因代码
- 数据传输：通常使用以太网或串口

## 7. 故障处理优先级原则

```
P0 安全类故障     → 立即停机，人工介入，不可自动恢复
P1 设备类故障     → 停机报警，排查后手动复位
P2 产品类故障     → 剔除产品，记录，程序继续
P3 通信类故障     → 重试机制（3次），超时后升级为P1
P4 软件类警告     → 记录日志，不停机，定期人工审查
```

## 8. 文档化原则

工程文档的最低要求：
- 系统架构图（物理连接 + 逻辑结构）
- IO 清单（地址 + 描述 + 有效电平）
- 程序注释（每个功能块的目的说明）
- 修改记录（日期 + 修改者 + 修改内容 + 原因）
- 调试记录（问题 + 解法 + 验证结果）
